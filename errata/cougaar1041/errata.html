Errata for Cougaar 10.4.1


The following infos/problems/workarounds were detected/posted after Cougaar 10.4.1 was released.

Issues:

  • When using Service Discovery and the new OPLAN mechanism, you must wait for all superior-subordinate relationships to by fully created, before sending the GLS task. To be safe, wait until all Nodes are idle (using top / Task Manager), before sending the task. A symptom of not waiting long enough is messages about "No Subsistence Provider" or other such message. See bug 2990.
  • Some agents are mis-configured in 1AD societies to do MaintainInventory, when they have no inventory to maintain. The result is un-confident MaintainInventory Tasks.
  • With the new OPLAN propagation mechanism, the OPLAN Edit Servlet must be loaded at every agent in the society, and each agent's oplan must be individually edited. Bug 3006
  • Some Java predicates in AggAgents are not correctly converted to XML. Bug 3005
  • Some AggAgent code uses == to compare Enums that have been serialized, and thus require .equals instead Bug 3003
  • WP bind/rebinds may squash each other, resulting in a logged error message like:

  • cougaar bug 3675: CircularQueue.toArray incorrectly copies elements, causing occasional NPE in topology code on startup. See bug for fix or ask for a patch release.
