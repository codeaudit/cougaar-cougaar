Errata for Cougaar 9.4


The following infos/problems/workarounds were detected/posted after the current version of Cougaar was built/released.

Issues:

  • The org.cougaar.core.persist.NonPersistable interface has been removed. This interface existed (in various packages) since an early implementation of persistence (over four years ago). It managed to survive through succeeding development despite that fact that it did nothing. Because it was attracting the attention of developers and being used (and failing to do what was expected), we decided to to remove it. This uncovered additional, non-functioning, uses. Use the org.cougaar.core.persist.NotPersistable interface to mark classes whose instances should not be saved in persistence snapshots.
  • CSMART Agent Insertion recipes exported from any release prior to 9.4 will give errors on importing into a 9.4 CSMART. These errors will be something like:
    ERROR [AgentInsertionRecipe] - Unknown property: Include Item 
Identification PG=true in component AGG-Agent
11:48:17,877 ERROR [AgentInsertionRecipe] - Unknown property: Include Org
Asset=true in component AGG-Agent

    Agent Insertion recipes in 9.4 must include an OrgAsset. So after importing your recipe, edit it, and fill out the newly required fields: ItemIdentification, TypeIdentification, etc, if you have not already done so. See bug #1594. See also below.

New Bugs:

  • Bug 1596: CSMART Agent Insertion recipe creates bad Agents. At runtime, you will get exceptions, because the OrgAssets it creates lack ClusterPG and ItemIdentifificationPG. This will be fixed for 9.4.1
